# Git: шпаргалка по основным командам

В этом репозитории я собираю команды Git.  
Здесь есть пояснения, примеры и комментарии, чтобы быстро вспомнить, что и зачем нужно.

---

## Краткая памятка
| Команда | Что делает |
|---------|------------|
| `git --version` | Проверить версию |
| `git config --global user.name "Имя"` | Настроить имя |
| `git config --global user.email "email"` | Настроить email |
| `git init` | Создать репозиторий |
| `git status` | Проверить состояние |
| `git add .` | Добавить все файлы |
| `git add файл.java` | Добавить конкретный файл |
| `git commit -m "сообщение"` | Сохранить изменения |
| `git log --oneline` | Посмотреть историю |

---

## Команды с пояснениями

### 1. git --version

**Что делает:**  
Проверяет, установлен ли Git, и показывает текущую версию.

```bash
git --version
```
**Пример вывода:**
```
git version 2.39.3
```
**Когда использовать:**  
Когда нужно убедиться, что Git установлен и работает.

---

### 2. git config
**Что делает:**  
Настраивает имя пользователя и email для коммитов.

```bash
git config --global user.name "AlexaJavaDev"
git config --global user.email "alexa@example.com"
```
**Что это даёт:**  
Каждый коммит будет подписан твоим именем и email.

**Проверить настройки:**  
```bash
git config --list
```

---

### 3. git init
**Что делает:**  
Создаёт новый Git-репозиторий в текущей папке.

```bash
git init
```
**Что появляется:**  
В папке создаётся скрытая папка `.git` — там хранится вся история изменений.

**Когда использовать:**  
Когда начинаешь новый проект и хочешь отслеживать изменения.

---

### 4. git status
**Что делает:**  
Показывает состояние репозитория: какие файлы изменены, какие добавлены, какие готовы к коммиту.

```bash
git status
```
**Пример вывода:**
```
On branch main
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
        modified:   README.md

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        newfile.java
```
**Когда использовать:**  
Постоянно, чтобы понимать, что происходит в репозитории.

---

### 5. git add
**Что делает:**  
Добавляет файлы в отслеживание (в staging area).

```bash
git add .           # добавить все файлы в текущей папке
git add README.md   # добавить конкретный файл
git add src/        # добавить всю папку src
```
**Что это даёт:**  
Файлы готовятся к коммиту.

**Когда использовать:**  
Перед каждым коммитом, чтобы выбрать, какие изменения сохранить.

---

### 6. git commit -m
**Что делает:**  
Сохраняет изменения с сообщением. Создаёт коммит.

```bash
git commit -m "Добавила новый метод в класс Main"
```
**Что это даёт:**  
Сохраняет снапшот текущего состояния проекта.

**Правила для сообщений:**

- Пиши в настоящем времени: "Добавила", "Исправила"
- Кратко и понятно
- Начинай с глагола

**Примеры:**

```bash
git commit -m "Добавила README"
git commit -m "Исправила ошибку в методе check"
git commit -m "Обновила учебный план"
```

---

### 7. git log --oneline
**Что делает:**  
Показывает историю коммитов в сокращённом виде (каждый коммит в одну строку).

```bash
git log --oneline
```
**Пример вывода:**
```
a1b2c3d Добавила README
e4f5g6h Исправила ошибку в методе check
i7j8k9l Обновила учебный план
```
**Когда использовать:**  
Когда нужно быстро посмотреть, какие изменения были сделаны и в каком порядке.

**Другие варианты:**

```bash
git log           # полная история с автором и датой
git log -n 3      # последние 3 коммита
git log --graph   # с визуализацией веток
```

---

## Мои заметки
- `git init` — делаю один раз в начале проекта
- `git status` — проверяю перед каждым коммитом
- `git add .` — добавляю все изменения за раз
- `git commit -m` — пишу понятное сообщение, чтобы потом не гадать, что я делала
- `git log --oneline` — смотрю, чтобы не потеряться в истории
